# PrepDeck

Flashcard-приложение для подготовки к собеседованиям с интервальным повторением (SM-2). PWA — работает офлайн.

## Запуск

Нужен любой локальный HTTP-сервер (Service Worker не работает через `file://`):

```bash
# Linux / macOS
python3 -m http.server 8000

# Windows
python -m http.server 8000
```

Открыть `http://localhost:8000` в браузере.

## Деплой

`localhost` доступен только на твоём компьютере. Чтобы открыть PrepDeck на телефоне или поделиться ссылкой, нужно выложить файлы на хостинг.

**GitHub Pages (бесплатно):**

1. Создай репозиторий на GitHub и запуши все файлы проекта
2. Settings → Pages → Source: `main` branch, папка `/ (root)` → Save
3. Через минуту приложение будет доступно по адресу `https://username.github.io/repo-name`

**Другие варианты:** Netlify, Vercel, Cloudflare Pages — все поддерживают статические сайты бесплатно. Достаточно подключить репозиторий — деплой произойдёт автоматически.

**Локальная сеть (без интернета):**

Если телефон и компьютер в одной Wi-Fi сети, можно открыть по IP компьютера:

```bash
# Узнай IP компьютера
hostname -I        # Linux
ipconfig getifaddr en0  # macOS

# Запусти сервер (уже знакомая команда)
python3 -m http.server 8000
```

Открой на телефоне `http://192.168.x.x:8000` (подставь свой IP). Но: Service Worker требует HTTPS или localhost, поэтому офлайн-режим и установка на домашний экран в локальной сети работать не будут. Для полноценного PWA нужен HTTPS — то есть хостинг.

## Установка на смартфон

После деплоя на хостинг (см. выше) PrepDeck можно установить на домашний экран как обычное приложение. Работает офлайн после первой загрузки.

**Android (Chrome):**

1. Открой приложение по ссылке в Chrome
2. Нажми на меню (⋮) в правом верхнем углу
3. Выбери «Установить приложение» или «Добавить на главный экран»
4. Подтверди установку

**iPhone / iPad (Safari):**

1. Открой приложение по ссылке в Safari (только Safari поддерживает PWA на iOS)
2. Нажми кнопку «Поделиться» (квадрат со стрелкой вверх)
3. Прокрути вниз и выбери «На экран Домой»
4. Нажми «Добавить»

После установки приложение запускается без адресной строки браузера и работает как нативное.

## Интерфейс

### Главный экран

- **Статистика** — всего карточек, изучено, осталось на сегодня
- **Прогресс** — процент изученных карточек
- **Темы** — фильтр по категориям (чипы). Минимум одна тема должна быть выбрана
- **Начать** — запускает сессию с карточками, которые пора повторить. Если все изучены — предлагает повторить всё

### Сессия

- Нажми на карточку, чтобы перевернуть и увидеть ответ
- Три кнопки оценки: **Знал** / **Сомневался** / **Не знал**
- Забытые карточки возвращаются позже в той же сессии
- Вверху — счётчик карточек и статистика ответов за сессию

### Результаты

После прохождения всех карточек — итоги сессии с количеством правильных, сомнительных и забытых ответов. Кнопка «Продолжить» появляется, если остались карточки на сегодня.

### Меню (&#9776;)

- **Загрузить карточки** — импорт JSON-файла (замена или добавление)
- **Экспорт данных** — бэкап всего (подробнее ниже)
- **Сбросить прогресс** — обнуление SM-2 данных (с подтверждением)

### Экспорт и импорт данных

PrepDeck хранит все данные в `localStorage` браузера. Это значит, что при очистке данных браузера или переходе на другое устройство прогресс теряется. Экспорт решает эту проблему.

**Что сохраняется в бэкап:**

- Все карточки (вопросы, ответы, категории)
- Прогресс SM-2 по каждой карточке (easeFactor, interval, repetitions, nextReview)
- Выбранные категории
- Дни занятий (streak)

Файл скачивается как `prepdeck-backup-2025-01-15.json` (с текущей датой).

**Зачем экспортировать:**

- Перенос на другой браузер или устройство
- Страховка перед очисткой данных браузера
- Резервная копия перед загрузкой новых карточек

**Где хранить бэкап:**

- Облако (Google Drive, Dropbox, iCloud)
- Локально на компьютере или флешке
- Отправить себе в мессенджер или на почту

**Как восстановить:**

Меню ☰ → Загрузить карточки → выбрать файл бэкапа. Приложение автоматически распознаёт формат (бэкап или обычные карточки) и предложит подтвердить восстановление. Текущие данные будут заменены.

### Форматирование в карточках

В тексте вопросов и ответов поддерживается подсветка кода:

- `` `инлайн код` `` — выделяется фоном и моноширинным шрифтом
- Блоки кода — оборачиваются в тройные обратные кавычки:

````
```js
const x = 42;
```
````

## Свои карточки

Создай JSON-файл и загрузи через меню ( Загрузить карточки):

```json
[
  {
    "question": "What is a closure in JavaScript?",
    "answer": "A function that retains access to variables from its outer (lexical) scope even after the outer function has returned. Used for data privacy, callbacks, and factory functions.",
    "category": "JavaScript"
  },
  {
    "question": "What is the CSS Box Model?",
    "answer": "Every element is a box consisting of: content → padding → border → margin (from inside out). box-sizing: border-box makes width/height include padding and border.",
    "category": "CSS"
  }
]
```

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `question` | string | да | Текст вопроса (до 5000 символов) |
| `answer` | string | да | Текст ответа (до 5000 символов) |
| `category` | string | нет | Категория для фильтрации (до 100 символов) |
| `id` | number/string | нет | Свой ID. Если не указан — генерируется автоматически из текста вопроса |

Максимум 5000 карточек. Размер файла — до 5 МБ.

## Генерация карточек через ИИ

Самый быстрый способ создать карточки — попросить ChatGPT, Claude или другую языковую модель. Скопируй промпт ниже, замени тему и получишь готовый JSON-файл. Промпт можно редактировать под свои запросы, но "Требования к формату", "Форматирование текста", "Пример одной карточки" лучше оставить как есть, так как код написан так что бы обрабатывать карточки именно в таком формате.

### Промпт

```
Сгенерируй набор флеш-карточек для подготовки к собеседованию на вакансию "Веб-разработчик" по теме: [ТЕМА].

Требования к формату:
- Верни ТОЛЬКО валидный JSON-массив, без пояснений до и после
- Каждая карточка — объект с полями: "question", "answer", "category"
- Вопросы и ответы на русском языке
- Категория — короткая строка для группировки (например: "React", "Hooks", "State")

Форматирование текста:
- Переносы строк: \n
- Инлайн-код: `код`
- Блок кода: ```язык\nкод\n```
- НЕ используй LaTeX-разметку ($...$), Markdown-заголовки (#) и жирный текст (**)

Пример одной карточки:
[
  {
    "question": "Что такое замыкание (closure) в JavaScript?",
    "answer": "A function that retains access to variables from its outer (lexical) scope even after the outer function has returned.\n\nExample:\n```js\nfunction counter() {\n  let count = 0;\n  return () => ++count;\n}\n```\n\nUsed for data privacy, callbacks, and factory functions.",
    "category": "JavaScript"
  }
]
```

### Как использовать

1. Скопируй промпт, замени `[ТЕМА]` на нужную (например: «React хуки», «SQL оптимизация», «Python asyncio»)
2. Отправь в ChatGPT / Claude / другую модель
3. Скопируй ответ (JSON-массив) в файл `cards.json`
4. В PrepDeck: меню ☰ → Загрузить карточки → выбери файл

Можно генерировать несколько наборов по разным темам и загружать через «Добавить к существующим».

## Алгоритм SM-2

SM-2 — это алгоритм интервального повторения, разработанный Петром Возняком в 1987 году для программы SuperMemo. Идея простая:

**Принцип**: чем лучше ты знаешь карточку, тем реже её нужно повторять.

## Как работает в PrepDeck

Каждая карточка хранит 4 параметра:

- **easeFactor** (2.5 по умолчанию) — насколько "лёгкая" карточка. Чем выше — тем быстрее растут интервалы
- **interval** — через сколько дней показать снова
- **repetitions** — сколько раз подряд ответил правильно
- **nextReview** — дата следующего показа

### После ответа

| Ответ | Что происходит |
|---|---|
| **Знал** (quality=5) | Интервал растёт: 1д → 6д → 15д → ... Карточка появляется всё реже |
| **Сомневался** (quality=3) | Интервал тоже растёт, но easeFactor снижается — рост будет медленнее |
| **Не знал** (quality=0) | Сброс: repetitions=0, карточка возвращается сразу |

### Пример пути карточки

```
День 1: "Что такое closure?" → Знал → следующий показ через 1 день
День 2: → Знал → через 6 дней
День 8: → Знал → через 15 дней
День 23: → Сомневался → через 15 дней (интервал не вырос)
День 38: → Знал → через 34 дня
...
```

Суть: карточки, которые ты знаешь хорошо, уходят далеко в будущее и не засоряют сессию. Трудные — крутятся часто.
